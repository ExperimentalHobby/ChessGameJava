## 開発・デバッグコマンド

### Java テスト・ビルド

```bash
# Java テスト全実行（Maven）
./mvnw.cmd test

# Swing のみビルド（Maven 不要）
build\build.bat

# Swing + JavaFX ビルド（Maven 使用）
build\build.bat --javafx

# Swing GUI 実行
java -cp target\classes com.chessgame.Main

# JavaFX GUI 実行
./mvnw.cmd javafx:run

# コンソール対話ゲーム実行
java -cp target\classes com.chessgame.InteractiveGame

# VS Code タスク（Ctrl+Shift+P → "タスクを実行"）
Build: Swing (dist)               # Swing のみをビルド
Build: JavaFX (dist)              # JavaFX のみをビルド
Build: Both (Swing + JavaFX)      # 両方を順番にビルド
```

### Python テスト

```bash
# Python AI テスト全実行
py -m unittest discover -s ai -p "test_*.py"

# 特定テストモジュールを実行
py -m unittest ai.test_engine -v

# perft テスト実行（move-gen 検証）
py -m unittest ai.test_engine_perft -v
```

## コーディング規約

### Java コード

#### 命名規則
- **クラス名**: PascalCase（`ChessGame`、`MoveValidator`）
- **メソッド名**: camelCase（`makeMove`、`getAvailableMoves`）
- **定数**: UPPER_CASE（`MAX_BOARD_SIZE`）
- **プライベートフィールド**: camelCase（`currentPlayer`）

#### Position の使用
- 座標は常に `Position` オブジェクトを使用
- DSL 形式を優先: `Position.of("e2")` / `Position.of(4, 4)`
- 生の行・列整数の保存は避ける

#### JavaDoc コメント
- **public / protected メソッド**: `@param` + `@return` を記載
- **テストメソッド**: 「何をすると何になるか（パス条件）」を明記

### Python コード

#### 命名規則
- **関数名**: snake_case（`get_flips`、`evaluate_position`）
- **クラス名**: PascalCase（`AlphaBetaAI`）
- **定数**: UPPER_CASE（`MAX_DEPTH`）

#### docstring
- **関数**: 1 行で収まる場合は 1 行 docstring
- **複雑な関数**: 複数行で引数・戻り値・例外を記述


## よくある落とし穴

1. **Board / 盤面の可変性** — 盤面を他のメソッドや履歴に渡す前に必ずコピーする。参照をそのまま保存すると Undo が壊れる。
2. **昇格ハンドリング** — ポーンが最終ランクに到達した際は必ず昇格プロンプトを表示し、手動で駒を選択させる。自動昇格は UI 体験を損なう。
3. **難易度4の実行時間** — 探索深さ 3 のエンジンは環境によって数秒要する。テストでは `depth=2` で代替するか、十分なタイムアウトを設定する。
4. **Position のインデックス** — `row 0 = ランク8`（最上段）、`col 0 = ファイル a` の規約を忘れない。
5. **FEN 表記法** — 難易度4の Python エンジンに渡す FEN は必ず正確に生成する。不正な FEN でエンジンが crash する場合がある。
