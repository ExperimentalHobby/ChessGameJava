# Swing / JavaFX UI 統一計画

## 1. 現状分析

### 1.1 Swing UI（安定版）
- **ゲームモード選択**: 横一列の5個ボタン（ChoiceDialog で実装）
  - "Human vs Human"
  - "Human vs AI (Easy)"
  - "Human vs AI (Medium)"
  - "Human vs AI (Hard)"
  - "Human vs AI (Expert)"
- **ウィンドウサイズ**: 840 × 550
- **コントロールパネル**: 右側に縦配置（New Game, Undo, Resign, Quit）
- **ボードサイズ**: 中央に占める割合が大きい
- **使用クラス**: `SwingChessGameFrame` + `SwingChessBoardPanel`

### 1.2 JavaFX UI（開発版）
- **ゲームモード選択**: ドロップダウンメニュー + OK/キャンセルダイアログ
  - ChoiceDialog を使用
  - リスト形式で選択
- **ウィンドウサイズ**: 未指定（デフォルト）
- **コントロールパネル**: 右側に配置（ボタン）
- **ボードサイズ**: Swing より小さく見える
- **使用クラス**: `ChessGameApp` + `ChessBoardView`

### 1.3 主な差異
| 項目 | Swing | JavaFX | 差異の影響 |
|-----|-------|--------|----------|
| ゲームモード選択 UI | 複数ボタン | ドロップダウン | ユーザーエクスペリエンス |
| ウィンドウサイズ | 固定 840×550 | デフォルト | 見た目、ボードサイズ |
| ボード占有率 | 大（左側にスペース） | 中程度 | 駒の見やすさ |
| コントロール配置 | 右側 | 右側 | ほぼ同じ |

---

## 2. 統一の目標

1. **ユーザーエクスペリエンスの統一**
   - 両バージョンで同じゲームモード選択UIを提供
   - 操作感を統一

2. **ウィンドウサイズの統一**
   - 両方とも 840 × 550 に統一
   - ボードサイズとコントロール配置も同じ

3. **コントロールパネルのレイアウト統一**
   - ボタン配置・サイズを統一
   - ボタンテキストの表記を統一

4. **ビジュアル統一**
   - 色合い（盤面の色など）を統一
   - フォント・文字サイズを統一

---

## 3. 実装戦略

### 3.1 ゲームモード選択 UI の変更

**Swing 側:**
- 現在の `ChoiceDialog` → そのまま（既に複数ボタン形式に見える）

**JavaFX 側:**
- `ChoiceDialog` → `Stage` + GridPane でカスタムダイアログを実装
- 5個ボタンを横一列に配置
- Swing と同じボタンテキスト・スタイルを使用

### 3.2 ウィンドウサイズの統一

**JavaFX 側:**
```java
// ChessGameApp.start() 内で
Scene scene = new Scene(root, 840, 550);  // Swing と同じサイズ
primaryStage.setResizable(false);  // リサイズ禁止
```

### 3.3 コントロールパネルの統一

**確認項目:**
- ボタンサイズ（幅・高さ）
- ボタン間隔
- フォントサイズ
- テキスト（"New Game", "Undo", "Resign", "Quit" に統一）

**JavaFX 側:**
- `ControlPanel` クラスで明示的にサイズを指定
- CSS でスタイルを定義（必要に応じて）

### 3.4 ビジュアル統一

**盤面の色:**
- Swing の色コードを取得
- JavaFX の SquareView で同じ色を使用

**駒のスタイル:**
- 現在：JavaFX は SVG, Swing も SVG（PieceImageGenerator）
- 一致性確認

## 4. 実装ステップ

### Phase 1: JavaFX ゲームモード選択 UI の改善（優先度: 高）
1. `ChessGameApp.showGameModeDialog()` を改修
2. `ChoiceDialog` の代わりにカスタムダイアログ + 5ボタンを実装
3. Swing と同じボタンテキスト・配置を使用
4. **テスト**: 各難易度を選択できることを確認

### Phase 2: ウィンドウサイズ・レイアウト統一（優先度: 高）
1. JavaFX の Scene サイズを 840 × 550 に統一
2. ウィンドウリサイズ禁止（`setResizable(false)`）
3. ボード・コントロール配置を Swing と揃える
4. **テスト**: UI が Swing と同じバランスで表示されることを確認

### Phase 3: ビジュアル詳細統一（優先度: 中）
1. 盤面の色を Swing と一致させる
2. ボタンスタイル（フォント・色）を統一
3. ステータスバーのレイアウト統一
4. **テスト**: 見た目の差異がないか確認

### Phase 4: テスト・ドキュメント更新（優先度: 中）
1. 両バージョンでゲーム開始〜終了まで動作確認
2. すべての難易度でAI選択・実行を確認
3. README.md に UI 統一に関する記述を追加（必要に応じて）

## 6. リスク・注意点

1. **JavaFX のカスタムダイアログ実装**
   - Stage / Scene の管理が複雑になる可能性
   - モーダルダイアログの実装確認が必要

2. **ウィンドウサイズ固定**
   - 小さいディスプレイではUIが切れる可能性
   - ただし Swing も同じため、既知の制限として扱う

3. **色・フォント の正確な統一**
   - Swing の色コードが明示的に記載されているか確認が必要
   - JavaFX CSS との変換が必要な場合がある

4. **ビジュアルテスト**
   - 手動で両バージョンを並べて比較する必要あり
